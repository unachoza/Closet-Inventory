The profile hub

Nothing To Wear · organizing a wardrobe, not building a social profile
Spec: planning/design/FIGMA_MAKE_PROMPT_profile.md · epic: E12
Core slice: hub (mobile + desktop) · locations · sharing

The profile is a hub of sections, each of which unlocks a capability — never one big form, never a completeness guilt-meter. Every section states its pillar (Inventory → Location → Status → Care → Share) and empty sections read as invitations. Flat surfaces, 12px cards, sentence case, two type weights. This core slice covers the hub on mobile + desktop, the location manager, and sharing & borrowers; onboarding + measurements/laundry details come in a later pass.

C2 — Sharing & borrowers
Viewers and loans, one honest page
Merges the Part Une viewer list (revoke = one tap) with active borrows (Part Deux). The closing lock-line restates the functional-vs-social divider: what friends can never see, in plain words.

i.Capabilities, not completeness

Empty sections invite ("Add measurements → unlocks Fits me now") — no percentages, no streaks, no badges. Users should feel organized, never judged.

ii.The functional–social divider is visible

"Only you" sections (measurements, laundry) wear the 🔒; shareable ones (identity, viewers) wear their pillar tag. The sharing page ends by naming what's never shared.

iii.Locations speak the closet's language

Kind-encoded borders on the manager rows match the card borders in the closet (E2) — the profile teaches the vocabulary the inventory already uses.

iv.Built on tables that exist

profiles, locations (per-user, RLS-scoped), closets + closet_members are live in Supabase (E12 reality check). This core slice needs one migration: locations.is_primary.

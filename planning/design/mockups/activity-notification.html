i.Notifications live in the profile

Locked 2026-07-10: no global bell, no tray, no push (a later, separate design). Activity is a hub section; the only ambient signal is the avatar dot. Calm beats immediate.

ii.Waiting ≠ unread

The model is "things needing my action" vs "things that happened." Waiting items pin to the top with inline verbs; Earlier items mark themselves read on view — the user never manages an inbox.

iii.The dot is the loudest we get

Terracotta, small, numberless in the nav. Counts appear only inside the hub card. Nothing is red; overdue is "a while," a denial is a dash — the palette carries the tone.

iv.One table, one writer

notifications (user_id, type, payload, read_at, acted_at), RLS owner-scoped, written in the same transaction as the borrow-request mutation that caused it — Activity can never disagree with the loan's actual state.

07 — Sharing panel
Every grant, visible and revocable
Links and viewers are separate lists (a link is a doorway; a viewer is a person). Open counts satisfy the audit need without analytics. Revocation checks on every request — no cached grants.

i.Privacy is server-side, three times over

isPrivate items are excluded from the Edge Function's teaser projection, excluded by the signed-in viewer RLS policy, and absent from viewer UI state. Blur is theatre; the payload is the boundary. Intimates default to private (US-4.4).

ii.The link is only the teaser key

128-bit token, stored hashed, expires 2h after first open, revocable, rate-limited against enumeration. Durable access never rides the URL — it requires Google sign-in and a closet_viewers row.

iii.Viewers get a lookbook

Name, photo, category, brand. Never prices, locations, wear history, or sync state. The viewer projection is a curated gallery — the owner's operational data stays home.

iv.Convert after value, not before

No wall on landing. Sign-in is asked for once the friend has seen twelve real pieces and wants the rest; the PWA install prompt and Gmail-import hook come only after the full closet renders.

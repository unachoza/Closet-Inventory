Engineering Quality Review

Mid/Sr Assessment

Overall grade: Strong. This codebase demonstrates engineering maturity well above what you'd expect from a solo project. Strict TypeScript with zero any types, 1,158 passing tests across 90 files, a clean Repository Pattern abstraction, and thoughtful security practices (DOMPurify XSS protection, in-memory-only OAuth tokens, localStorage cleanup of legacy credentials). The architecture decisions are defensible and the test coverage is real, not ceremonial.

Strengths

  • Type safety: Zero any types in source. Strict mode + noUnusedLocals + noFallthroughCases. Auto-generated Supabase types used correctly.
  • Repository pattern: Clean ClosetRepository interface with Local, Supabase, and Synced implementations. Backend swap is a one-line change.
  • Security posture: Gmail tokens in-memory only. DOMPurify with link sanitization. Legacy credential cleanup on mount. Gitleaks CI. RLS proven with real tests.
  • Test quality: XSS-specific tests, animation regression tests, sync failure lifecycle tests, integration tests. Not just happy-path coverage.
  • Domain modeling: FashionParser is a proper module (17 attribute maps, normalizers, inference engine), not scattered utils.

Issues to Address Before Launch

  • Data integrity: upsertMaterials does delete-then-insert without a transaction. If the insert fails, materials are lost. Wrap in an RPC or use ON CONFLICT.
  • N+1 on import: importItems fires a separate delete+insert pair per item for materials. A 50-item import is 100+ queries. Batch this.
  • Keyboard accessibility: Card.tsx uses onClick on a div without role="button", tabIndex, or keyboard handlers. Close button in EditItemView is an <img onClick>. These are WCAG failures.
  • No focus trapping: Card detail modal doesn't trap focus. Tab can reach content behind the overlay.

Recommendations (Post-Launch)

  • Extract email parser: parseProductsFromEmail.ts is 4,000+ lines. Each retailer strategy should be its own module.
  • Memoize Fuse index: useFuzzySearch creates a new Fuse instance on every search call. Memoize against the items array.
  • Code splitting: No React.lazy anywhere. GmailImport, FabricCare, Onboarding should lazy-load.
  • Unify care type: Currently string | string[], forcing defensive normalization everywhere. Standardize to string[].
  • Missing Supabase repo tests: Only local + synced repos have tests. The Supabase repo's row mapping and error paths are untested.
